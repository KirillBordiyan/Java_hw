Hw5 - MVC, информационная система сотрудников (на примере программистской организации,
 ее структура представлена в этом же слое)

Выбран паттерн классическая MVC, тк:
 -наиболее проста и понятна в реализации;
 -все границы четко выстроены;
 -нет излишних пересечений сущностей;

Папка model содержит в себе разделения на сущности и сервис.
В сущности описана наименьшая единица - работник(сотрудник)
в сервисе происходит управление новыми сущностями (добавление, удаление,
            получение всех сотрудников и поиск по id конкретного сотрудника);

Папка controller имеет EmployeeController, который позволяет пользователю выбирать соответствующие действия.

Папка view имеет EmployeeView, с помощью которого происходит вызов соответствующих методов и их преобразование;

Бизнес-процесс:
    при запуске программы у нас заведомо есть CEO, в подчинении которого пока никого нет;

    добавляем несколько сотрудников, которым определяем руководителей и подчиненных;

    в консоли после каждого действия будет конечный результат, чтобы можно было проверить верность результата;

    при удалении сотрудника также надо учитывать, что он больше не руководитель и не чей-либо подчиненный,
    поэтому у остальных сотрудников его данные полностью стираются;

Для полного использования можно добавить изменение руководителя у конкретного сотрудника и автосортировку по id
(это, если получится, добавлю в следующее дз)
