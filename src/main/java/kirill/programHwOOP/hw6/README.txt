SRP - применялся для классов:
        ->Letter
        ->Package
        ->Client

OCP - применяется в:
        ->наследование Email от Letter, в Email можно добавлять свои методы

LSP - применяется в:
        ->система не сломается, если отправлять любой из 3 видов посылок

ISP - применяется в:
        -> для электронных писем имеется EmailInterface, который не видят пользователи
            тк он нужен только для электронных писем

DIP - применяется в:
        -> от интерфейса SendingOption можно создать любой экземпляр класса для отправки
